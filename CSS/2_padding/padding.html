Physical Padding

These padding consists of padding(top, right, bottom, left).

Logical Padding

padding-inline: applies padding on both the start and end of the inline axis

Logical Padding

padding-block: applies padding on both the start and end of the block axis (usually top and bottom in horizontal writing modes).

Logical Padding

padding-inline-start: applies padding at the start of the inline axis (left in LTR, right in RTL)

padding-inline-end: applies padding at the end of the inline axis (right in LTR, left in RTL)

Difference between logical and physical padding

Physical padding

  • These properties are fixed in direction and don't change based on the text direction or writing mode of the document.
  • Ideal for layouts where the direction (left, right, top, bottom) is constant and does not need to adapt to different languages or writing directions.

Logical padding

  • These properties adapt to the writing mode and text direction (LTR or RTL) of the document.
  • Ideal for multilingual layouts that need to adjust to different writing directions without changing CSS rules for each direction.
